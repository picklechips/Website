BRACELit - C++, SDL2, Arduino

The ability to remember whether or not you've completed simple tasks such as locking the door or shutting the windows becomes difficult for those with alzheimer's or dementia and can often become a strong source of anxiety.

BraceLit is a simple and fashionable solution in the form of wearable tech that acts as a reminder service for such tasks.

Working alongside three of my friends, we were able to create a working prototype as well as explore the potential market and industry. The idea was to create a discrete and incredibly simple device useable by those strongly affected.
A simple piece of hardware is attached to a desired task, such as the lock on a door. When the door locks, an icon on the bracelet is updated to visually demonstrate that the door is locked. When the door unlocks, the icon is again updated accordingly. This allows the user to simply verify whether or not the door has been locked, allowing them to continue their day with their mind at ease.

Using an Arduino, we were able to create a working demonstration using C++ and SDL2.

SHARK Billiards Locator - Java, FireBase

Created at Hack the North in 2016 by my friend and I, SHARK Billiards Locator is a mobile application for android devices which allows users to find, review, and share local pool tables. As a fan of Billiards games myself, it's not always easy to find nearby pool tables other than by word of mouth. Hence, SHARK Billiards locator.

List view of tables

Submitting a location

Sort the list

The idea is to allow users to look up pool tables closest to them and filter them based on rating and cost. If a user happens to stumble upon a non well-known table, they can also submit the table to the database to allow other users to then find it.

The application was created using Java, utlizing google's Firebase as an online database.

A* Pathfinder - C++, OpenGL

A* is a lightweight pathfinding algorithm for predicting and calculating the most efficient path around obstacles from point A to point B.

I implemented the algorithm in 3D Space using C++ and OpenGL. After creating a 3D scene with replaceable, moveable obstacles, the user can click somewhere in the scene and a Cube would navigate around the obstacles in order to reach the destination.

FRAMEWORK

SchoolYard is a personal games framework I'm building on top of C++ and SDL2, with the respective game SchoolYard to be built using it. In it's current state, schoolyard can load scenes - including data such as collision - from CSV files and render the map to the screen. It also handles player movement, animation, and interaction.
